# 1. 実施ルール

- SQL作成完了後に実施すること
- **自己チェック → レビュー → 修正 → 再確認**
- 該当なしは「N/A」
- 問題ありは備考へ記載
- **UPDATE / DELETE は必ずレビュー対象**

判定：

| 記号 | 意味 |
|---|---|
| ○ | 問題なし |
| × | 要修正 |
| △ | 要確認 |
| N/A | 対象外 |
