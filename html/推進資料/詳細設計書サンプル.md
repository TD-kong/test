## 6. シーケンス設計（処理フロー）

### 6.1 申請登録処理

```mermaid
sequenceDiagram
    actor User
    participant Controller
    participant Service
    participant Dao
    participant DB

    User->>Controller: 申請登録
    Controller->>Service: registerApply()
    Service->>Dao: insertApply()
    Dao->>DB: INSERT
    DB-->>Dao: OK
    Dao-->>Service: result
    Service-->>Controller: result
    Controller-->>User: 完了通知
```

---

### 6.2 承認処理フロー

```mermaid
sequenceDiagram
    actor Approver
    participant Controller
    participant Service
    participant Dao
    participant DB

    Approver->>Controller: 承認操作
    Controller->>Service: approveApply()
    Service->>Dao: updateStatus()
    Dao->>DB: UPDATE
    DB-->>Dao: OK
    Dao-->>Service: result
    Service-->>Controller: result
    Controller-->>Approver: 完了
```

## 7. SQL設計

### 7.1 テーブル定義

#### ■ T_APPLY（申請テーブル）

| カラム名 | 型 | 制約 | 説明 |
|----------|----|------|------|
| APPLY_ID | VARCHAR2(20) | PK | 申請ID |
| USER_ID | VARCHAR2(20) | NOT NULL | 申請者ID |
| CONTENT | VARCHAR2(500) |  | 申請内容 |
| STATUS | VARCHAR2(10) |  | ステータス |
| CREATE_DATE | DATE |  | 作成日 |

---

### 7.2 DDL

```sql
CREATE TABLE T_APPLY (
    APPLY_ID VARCHAR2(20) PRIMARY KEY,
    USER_ID VARCHAR2(20) NOT NULL,
    CONTENT VARCHAR2(500),
    STATUS VARCHAR2(10),
    CREATE_DATE DATE
);
```

---

### 7.3 SQL（CRUD例）

#### ■ 申請登録

```sql
INSERT INTO T_APPLY (
    APPLY_ID,
    USER_ID,
    CONTENT,
    STATUS,
    CREATE_DATE
) VALUES (
    ?, ?, ?, 'NEW', SYSDATE
);
```

---

#### ■ 申請検索

```sql
SELECT *
FROM T_APPLY
WHERE USER_ID = ?
  AND STATUS = ?;
```

---

#### ■ 承認更新

```sql
UPDATE T_APPLY
SET STATUS = 'APPROVED'
WHERE APPLY_ID = ?;
```
