## 2. 目的

本書は、本番リリース実施可否を判断するためのチェック項目を定義し、
品質・リスク・運用準備状況を統合的に評価することを目的とする。

---

## 3. 判定区分

| 判定 | 意味 |
|------|------|
| Go | リリース実施可能 |
| No-Go | リリース不可（課題あり） |

---

## 4. 総合判定基準

- 重大障害（Severity A）が0件であること
- 未解決バグが残っていないこと
- 業務シナリオテストが全件完了していること
- 運用準備が完了していること

### 11.3 条件付きリリース

- 軽微バグのみ残存
- 回避策が明確
- 業務影響なし
